AIDL 是干嘛的 ?
Android API 中给出

Note: Using AIDL is necessary only if you allow clients from different applications to access your service for IPC and want to handle multithreading in your service. If you do not need to perform concurrent IPC across different applications, you should create your interface by implementing a Binder or, if you want to perform IPC, but do not need to handle multithreading, implement your interface using a Messenger. Regardless, be sure that you understand Bound Services before implementing an AIDL.

即不同的Application 访问服务，并且需要多线程的时候使用AIDL，如果是多进程的单线程可以用Messenger 来来实现

  • 定义AIDL接口
          创建一个 xx.aidl 文件
          aidl文件里面的写法和java写法一样，但是只能定义接口，返回值和参数值默认支持基本数据类型和String，CharSequence，List ，Map（List和Map的元素必须是受到支持的数据类型，或者是其他AIDL生成的接口或者是实现 parcelables 接口的），如果需要导入其他类型的参数，需要在开头import他们
          定义接口需要注意
               1：方法采用0个或多个参数，然后返回一个值或返回void 
               2：在AIDL中的注释会自动添加到自动生成的java类中
               3：如果非上诉红色字体的类型需要指定方向 （in:客户端设置 ,out:服务端设置 ,inout:两端都可以，下面会详细讲解）
               4：AIDL只支持写方法，不能暴露静态的字段在AIDL中
           例子
               1.自定义类型
                    首先创建一个 类型.aidl 文件
                    
package blq.ssnb.aidl;

parcelable MInfo;
                    这样就可以在其他的接口中导入这个自定义类了（还需要实现MInfo这个类，且实现Parcelable接口）
               2.自定义接口
                  创建一个 自定义接口名.aidl 文件

// ICallBack.aidl
package blq.ssnb.aidl;

// Declare any non-default types here with import statements

interface ICallBack {
    void sendMsg(String msg);
}


// IExecuteFunction.aidl
package blq.ssnb.aidl;

// Declare any non-default types here with import statements
import blq.ssnb.aidl.ICallBack;这个是在上面ICallBack.aidl 接口
import blq.ssnb.aidl.MInfo;//这个就是上面创建的其他类型

interface IExecuteFunction {
     //返回MInfo 对象，参数 前面带 可以带in、out、inout
     //in out inout 下面会详细讲解
     //ICallBack 不是数据类型是 接口所以不用加（只能是in）
    MInfo getServiceInfo(in MInfo info , ICallBack callBack);
}

  • 实现接口
     
// 创建一个service 用来创建接口通过服务绑定向客户端提供接口
public class MyService extends Service {
    public MyService() {
    }

    @Override
    public IBinder onBind(Intent intent) {
        return mBinder;
    }

//系统会默认对.aidl文件生成相应的java 类，且创建内部类Stub 用来实现对应接口 
    private IExecuteFunction.Stub mBinder = new IExecuteFunction.Stub() {
        @Override
        public MInfo getServiceInfo(MInfo info, ICallBack callBack) throws RemoteException {
            Log.e("MyService","mBinder-getServiceInfo=======>>>>>>>>>>>"+Thread.currentThread());
            Log.e("MyService","mBinder-getServiceInfo-info==========>>>>>>>>>>"+(info==null?"null":info.toString()));
            callBack.sendMsg("ssnb");
            info.setName("mSSNB");
            info.setAge(24);
            Log.e("MyService","mBinder-getServiceInfo-info==========>>>>>>>>>>"+info.toString());
            return info;
        }
    };
}

//调用方法入口
private void connectionServie(){
    Intent intent = new Intent();
    intent.setAction("blq.ssnb.action");//service的Action
    startService(intent);//启动服务
    bindService(intent,connection,BIND_AUTO_CREATE);//绑定服务
}
private IExecuteFunction executeFunction;//接口

ServiceConnection connection = new ServiceConnection() {
    @Override
    public void onServiceConnected(ComponentName name, IBinder service) {
        Log.e("connection","onServiceConnected:"+name);
        executeFunction = IExecuteFunction.Stub.asInterface(service);//绑定成功后从服务端获取接口
        if(executeFunction!=null){
            sendInfo();
        }
    }
    @Override
    public void onServiceDisconnected(ComponentName name) {
        Log.e("connection","onServiceDisconnected:"+name);
    }
};

//这里将调用接口
static int i = 0;
private void sendInfo() {
    MInfo info = new MInfo();
    info.setName("ssnb:"+i);
    info.setAge(20+i);
    i++;

    try {
        MInfo serviceInfo = executeFunction.getServiceInfo(info,callBack);
        unbindService(connection);
        Log.e("MainActivity","info:"+info.toString());
        Log.e("MainActivity","serviceInfo:"+serviceInfo.toString());
    } catch (RemoteException e) {
        e.printStackTrace();
    }
}

ICallBack.Stub callBack = new ICallBack.Stub() {
    @Override
    public void sendMsg(String msg) throws RemoteException {
        Toast.makeText(MainActivity.this,"服务端返回:"+msg,Toast.LENGTH_SHORT).show();
    }
};

in、out、inout 的理解
     基本数据类型、String和CharSequence 默认是in 标识，其他的类型需要添加 in，out ，inout 标识来区分
     
1.事先定义接口
interface IExecuteFunction {

    MInfo getServiceIn(in MInfo info);
    MInfo getServiceOut(out MInfo info);
    MInfo getServiceInOut(inout MInfo info);

}

2.服务端代码

三个接口同一调用该方法
prx("in",info,1111);
prx("out",info,2222);
prx("inout",info,3333);
private MInfo prx(String tag ,MInfo info,int age){
    L("-----"+tag+"-start--------");
    printInfo(tag+"-bef",info);
    info.setName("我是"+tag+"标识");
    info.setAge(age);
    printInfo(tag+"-aft",info);

    MInfo newInfo = new MInfo();
    newInfo.setName("我是"+tag+"标识");
    newInfo.setAge(age);
    printInfo(tag+"-new",newInfo);
    L("-----"+tag+"-end--------");
    return newInfo;
}

3.in
数据由客户端提供，服务端可以接受客户端的数据，在服务端改变该数据不会影响客户端的数据
例：

String msg ;
MInfo inInfo = new MInfo();
if(indexIn%2==0){
    msg ="有数据";
    inInfo.setName("客户端in");
    inInfo.setAge(7777);
}else{
    msg ="无数据";
}
indexIn++;
printInfo("in-"+msg+"-bef",inInfo);
MInfo newInfo = executeFunction.getServiceIn(inInfo);
printInfo("in-"+msg+"-aft",inInfo);
printInfo("in-"+msg+"-return",newInfo);

4.out
客户端传入对象，数据由服务端填写，服务端的改变会直接影响到客户端的数据
例：

String msg ;
MInfo outInfo = new MInfo();
if(indexOut%2==0){
    msg ="有数据";
    outInfo.setName("客户端out");
    outInfo.setAge(8888);
}else{
    msg ="无数据";
}
indexOut++;
printInfo("out-"+msg+"-bef",outInfo);
MInfo newInfo = executeFunction.getServiceOut(outInfo);
printInfo("out-"+msg+"-aft",outInfo);
printInfo("out-"+msg+"-return",newInfo);

容易碰到的坑
1.Android Studio 或eclipse 没有编译过在代码里是找不到对应接口类的
2.在Android Studio 中创建aidl文件，系统会自动创建一个aidl文件，将文件放入该文件夹下
默认结构
 
从而导致编译的时候找不到类

解决方法 build 中加入
android {

sourceSets {
    main {
        manifest.srcFile 'src/main/AndroidManifest.xml'
        java.srcDirs = ['src/main/java', 'src/main/aidl']
        resources.srcDirs = ['src/main/java', 'src/main/aidl']
        aidl.srcDirs = ['src/main/aidl']
        res.srcDirs = ['src/main/res']
        assets.srcDirs = ['src/main/assets']
    }
}
}
